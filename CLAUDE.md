# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Single-page marketing/blog site for **Letícia Pais, psicóloga (CRP 10/11502)** — Portuguese-language content, TCC (terapia cognitivo-comportamental) theme. Generated by Hostinger Horizons / app-preview, then exported. React 18 + Vite 7 SPA, no backend, no tests.

## Commands

Node 22 (`.nvmrc`). `node_modules` is NOT installed in a fresh checkout — run install first.

```bash
npm install          # required before anything else
npm run dev          # Vite dev server on http://localhost:3000 (host ::)
npm run build        # runs tools/generate-llms.js, then vite build -> dist/
npm run preview      # serve the built dist/ on :3000
npm run lint         # eslint . --quiet
```

No test runner is configured. There is no single-test command.

## Architecture

- **Entry:** `src/main.jsx` → `src/App.jsx`. `App.jsx` wraps everything in `ThemeProvider` then `BrowserRouter`.
- **Routes** (all in `App.jsx`): `/` → `pages/Home.jsx`, `/blog` → `pages/Blog.jsx`, `/blog/:postId` → `pages/BlogPost.jsx`.
- **Home** is a one-pager: composes section components from `src/components/` (`Header`, `Hero`, `About`, `Services`, `BlogSection`, `FAQ`, `Contact`, `Footer`). Edit these to change the landing page.
- **Blog content is hardcoded** in `src/lib/blogData.jsx` — an array of post objects with raw HTML strings in `content`. No CMS. Add/edit posts there; `Blog.jsx` lists them and `BlogPost.jsx` renders one by `id`.
- **Theme:** `src/context/ThemeContext.jsx` — light/dark via a class on `<html>`, persisted to `localStorage('theme')`, default `light`. Use the `useTheme()` hook.
- **UI kit:** `src/components/ui/` is shadcn/ui (new-york style, JSX not TSX — see `components.json`) built on Radix + CVA + tailwind-merge. Combine classes with `cn()` from `src/lib/utils.js`. Most of these files are unused by the actual site; only pull from them as needed.
- **SEO:** each page sets `<Helmet>` (react-helmet) with title/description. `tools/generate-llms.js` scrapes those Helmet tags at build time to emit `public/llms.txt`.

### Conventions

- Import alias `@/` → `src/` (configured in `vite.config.js`, `jsconfig.json`, and the eslint import resolver). Use it everywhere.
- Animations use `framer-motion`; icons use `lucide-react`. Scroll-reveal helper: `src/hooks/useInView.js`.
- Styling is Tailwind only (config `tailwind.config.js`, CSS vars in `src/index.css`). No CSS modules.
- ESLint is deliberately lenient: `no-unused-vars`, `react/prop-types`, and react-in-scope rules are **off**; `no-undef` and `import/no-self-import` are the main hard errors. Don't expect lint to catch unused code.

### Horizons editor plugins (dev only)

`vite.config.js` loads custom plugins from `plugins/` (`visual-editor`, `selection-mode`, iframe route restoration) **only when `NODE_ENV !== 'production'`**. They power the visual editor inside the app-preview iframe and inject `postMessage`-based error/console forwarding to a parent frame. They have no effect on the production build and can be ignored for normal feature work.

## Known cruft / things that don't work

- **`src/hooks/use-toast.js` vs `src/components/ui/use-toast.js`** — two toast hook copies exist (shadcn duplication). Confirm which one a component imports before editing.
- Build script swallows failures: `node tools/generate-llms.js || true` — llms.txt generation can silently fail without breaking the build.
